Contexte et Objectif

La plateforme web de mise en relation, développée avec Spring Boot (backend) et Angular (frontend), nécessite un environnement d’hébergement capable de garantir des performances optimales, une haute disponibilité et une sécurité renforcée.

Cette proposition vise à mettre en place une infrastructure Cloud moderne et scalable, facilitant le déploiement continu, la maintenance, et l’évolution future de votre application.

Architecture Technique Proposée

L’hébergement repose sur une architecture découplée entre le frontend Angular et le backend Spring Boot, garantissant une gestion efficace des performances, de la sécurité et de la scalabilité.

Frontend Angular

Déployé sur un CDN mondial (ex : AWS S3 + CloudFront ou Firebase Hosting) pour un chargement rapide.

Backend Spring Boot

Conteneurisé avec Docker, déployé sur un service Cloud (AWS ECS, Render ou Azure App Service).

Base de Données

MySQL hébergé sur RDS/Azure Database avec réplication et sauvegardes automatiques.

Authentification

Gestion des utilisateurs via JSON Web Token (JWT), intégrée au backend Spring Boot pour l’authentification et l’autorisation sécurisées.

Architecture Cloud Détaillée

Découvrez plus en détail la conception technique de l’infrastructure d’hébergement proposée :

Frontend Angular (AWS S3 + CloudFront) ▼

L’application Angular est hébergée sur AWS S3 (stockage statique) et diffusée via CloudFront (CDN mondial) garantissant des temps de chargement inférieurs à 100 ms dans toute l’Europe.

  • HTTPS & HTTP/2 activés
  • Cache intelligent via CloudFront
  • Redondance multi-régions
Backend Spring Boot (AWS ECS / Elastic Beanstalk) ▼

Le backend Java Spring Boot est conteneurisé avec Docker et déployé sur Elastic Beanstalk ou ECS (Elastic Container Service). Cette approche assure scalabilité, redémarrage automatique et intégration continue.

  • Autoscaling selon la charge CPU/RAM
  • Intégration CI/CD via GitHub Actions
  • Logs et monitoring via CloudWatch
Base de Données (AWS RDS MySQL) ▼

La base MySQL est hébergée sur AWS RDS avec sauvegardes automatiques, réplication multi-zone et chiffrement au repos (KMS).

  • Réplication automatique (Multi-AZ)
  • Sauvegardes quotidiennes et restauration rapide
  • Connexion sécurisée (SSL/TLS)
Authentification & Sécurité (Keycloak + AWS IAM) ▼

Keycloak gère l’authentification OAuth2/OpenID. Côté AWS, les accès sont protégés par des rôles IAM et des politiques restreintes.

  • Single Sign-On (SSO)
  • Chiffrement des tokens JWT
  • Gestion des rôles par IAM (Identity Access Management)

Services AWS Utilisés

L’infrastructure repose sur les services clés d’AWS pour garantir performance, sécurité et flexibilité :

Amazon EC2
Instances virtualisées pour exécuter le backend.

EC2 permet de lancer des serveurs virtuels à la demande. Idéal pour le déploiement d’applications backend évolutives avec tarification à l’usage.

Amazon S3
Stockage d’objets pour fichiers statiques.

S3 offre un stockage sécurisé, durable et accessible pour les images, vidéos et documents statiques, avec réplication inter-région et contrôle d’accès intégré.

Amazon RDS
Gestion de bases de données relationnelles.

RDS simplifie la configuration, la gestion et la mise à l’échelle des bases de données comme MySQL ou PostgreSQL. Sauvegardes automatiques et haute disponibilité incluses.

AWS IAM
Gestion des accès et des identités.

IAM permet de définir précisément les permissions utilisateurs et services, garantissant une sécurité granulaire au sein de l’environnement cloud.

Amazon CloudFront
Réseau de diffusion de contenu (CDN).

CloudFront accélère la diffusion des contenus vers les utilisateurs finaux, grâce à un cache géolocalisé, un chiffrement TLS/SSL intégré et une disponibilité mondiale.

AWS Lambda
Exécution de fonctions serverless.

Lambda permet d’exécuter du code sans gérer de serveurs. Facturation uniquement à la durée d’exécution, idéal pour les tâches automatisées.

Amazon VPC
Réseau privé virtuel sécurisé.

VPC permet d’isoler les ressources dans un environnement réseau privé avec contrôle total sur les sous-réseaux, pare-feu et passerelles.

Amazon CloudWatch
Supervision et métriques.

CloudWatch collecte et analyse les métriques d’application et d’infrastructure, facilitant la détection proactive des anomalies et l’optimisation des performances.

Amazon Route 53
Gestion DNS et routage intelligent.

Route 53 assure la résolution DNS rapide, la redirection de trafic et l’équilibrage géographique pour une haute disponibilité mondiale.

Elastic Load Balancer
Répartition automatique du trafic.

ELB distribue intelligemment les requêtes entre plusieurs instances EC2, assurant une meilleure tolérance aux pannes et des performances optimales.

Régions et Zones d’Hébergement

Les services AWS sont répartis dans plusieurs régions géographiques, chacune disposant de zones de disponibilité (AZ) indépendantes. Cela garantit à FibemFrance une haute disponibilité, une redondance géographique et une latence minimale.

🇫🇷 Région Europe (Paris)

Hébergement principal des serveurs FibemFrance

Les serveurs principaux seront déployés dans la région eu-west-3 (Paris). Cette région offre trois zones de disponibilité (A, B, C) garantissant la tolérance aux pannes et une connexion rapide pour les utilisateurs francophones.

🇩🇪 Région Europe (Francfort)

Zone de secours pour redondance

Une réplication automatique des données critiques sera configurée vers eu-central-1 (Francfort), garantissant la continuité de service même en cas d’incident sur la région principale.

🌍 Architecture Multi-AZ

Disponibilité et tolérance aux pannes

L’architecture repose sur un modèle Multi-AZ (Availability Zones), où chaque instance EC2 et base de données RDS est répliquée sur plusieurs zones. Cela assure la résilience du système en cas de défaillance matérielle ou de maintenance planifiée.

🔐 VPN et Accès Sécurisé

Connexion privée entre le réseau interne et AWS

L’accès aux serveurs AWS se fait via un VPN privé basé sur AWS Client VPN. Seuls les administrateurs autorisés peuvent se connecter, garantissant la confidentialité et la sécurité du réseau.

L’intranet permet d’ajouter, modifier ou désactiver des comptes administratifs et utilisateurs. Des rôles et permissions spécifiques sont définis (admin, gestionnaire, support…) afin de garantir une gouvernance claire et une sécurité renforcée.
